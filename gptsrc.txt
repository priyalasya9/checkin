# 🟢 STEP 5: Add Lambda Code

### 👉 Replace code with this:

```python
import boto3
from uuid import uuid4

def lambda_handler(event, context):
    s3 = boto3.client("s3")
    dynamodb = boto3.resource('dynamodb')

    if 'Records' in event:
        for record in event['Records']:
            bucket_name = record['s3']['bucket']['name']
            object_key = record['s3']['object']['key']
            size = record['s3']['object'].get('size', -1)
            event_name = record.get('eventName', 'Unknown')
            event_time = record.get('eventTime', 'Unknown')

            table = dynamodb.Table('my-table')

            table.put_item(
                Item={
                    'id': str(uuid4()),
                    'Bucket': bucket_name,
                    'Object': object_key,
                    'Size': size,
                    'Event': event_name,
                    'Time': event_time
                }
            )
    else:
        print("No Records found")
```

---

```json
{
  "Version": "2012-10-17",

      "Action": "SNS:Publish",
      "Resource": "SNS-TOPIC-ARN",
      "Condition": {
        "ArnLike": {
          "aws:SourceArn": "S3-BUCKET-ARN"
        },
        "StringEquals": {
          "aws:SourceAccount": "YOUR-ACCOUNT-ID"
        }
      }
    }
  ]
}
```

# ✅ WEEK 13 — IAM ROLES (FULL STEP-BY-STEP)

📄 Source:  (expanded + simplified)

---

# 🔷 OVERVIEW (Understand FIRST)

IAM Role = **temporary permissions for services (no passwords/keys)**

```text
EC2 → assumes role → gets temporary access → uses AWS services
```

👉 Example: EC2 accessing S3 **without storing keys**

---

# 🟢 WHY ROLES? (IMPORTANT)

❌ IAM User → needs access keys (not secure)
✅ IAM Role → automatic temporary credentials (secure)

---

# 🟢 STEP 1: CREATE IAM ROLE

---

## 👉 Do this:

1. Open AWS Console
2. Search → **IAM**
3. Click **Roles**
4. Click **Create role**

---

## STEP 1.1: Trusted Entity

* Select → **AWS Service**

---

## STEP 1.2: Use Case

* Select → **EC2**

👉 Click **Next**

---

# 🟢 STEP 2: ADD PERMISSIONS

1. Search:

   * `AmazonS3FullAccess`

2. Select it

👉 Click **Next**

---

# 🟢 STEP 3: NAME ROLE

Fill:

* Role name → `EC2-S3-Role`

👉 Click **Create role**

---

# 🟢 STEP 4: ATTACH ROLE TO EC2

---

## 👉 Steps:

1. Go to **EC2 Dashboard**
2. Click **Instances**
3. Select your instance

---

### Attach Role:

1. Click:

   * Actions → Security → Modify IAM role

2. Select:

   * `EC2-S3-Role`

3. Click **Update**

---

# 🟢 STEP 5: VERIFY (VERY IMPORTANT)

---

## 👉 Connect to EC2

Use EC2 connect / SSH

---

## Run:

```bash
aws s3 ls
```

✅ Works → shows buckets

---

## Run:

```bash
aws ec2 describe-instances
```

❌ Fails → Unauthorized

---

### 👉 WHY?

Because:

* Role has **only S3 permission**
* No EC2 permission

📌 This exact explanation is expected in viva

# 🔥 IMPORTANT CONCEPTS (EXAM GOLD)

---

## ✔ IAM Role

Temporary permission for AWS services

---

## ✔ No Access Keys Needed

AWS automatically provides credentials

---

## ✔ Least Privilege

Only required permissions

---

## ✔ Role vs User

| IAM User      | IAM Role         |
| ------------- | ---------------- |
| Permanent     | Temporary        |
| Password/Keys | No keys          |
| Human login   | Used by services |
