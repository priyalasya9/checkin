Analysing Android App Permissions and Mobile Traffic
Procedure
1. Create Android Virtual Device

1.1 Install Android Studio
1.2 Open AVD Manager
1.3 Click “Create Virtual Device”
1.4 Select device (Pixel 2 / Pixel 9)
1.5 Complete setup

2. Start Emulator with Proxy

2.1 Open Command Prompt
2.2 Navigate to emulator directory:

cd C:\Users\Admin\AppData\Local\Android\Sdk\emulator

2.3 List available AVDs:

emulator -list-avds

2.4 Start emulator with proxy:

emulator -avd Pixel_9 -http-proxy http://10.0.2.2:8080
3. Configure Proxy in Emulator

3.1 Open Settings
3.2 Go to Network & Internet → Internet
3.3 Select connected network
3.4 Click Edit → Advanced options
3.5 Change Proxy to Manual
3.6 Enter Hostname: 10.0.2.2
3.7 Enter Port: 8080
3.8 Click Save

4. Configure Proxy using ADB (Alternative)

4.1 Open Command Prompt
4.2 Navigate to platform-tools:

cd C:\Users\Admin\AppData\Local\Android\Sdk\platform-tools

4.3 Check connected devices:

adb devices

4.4 Set proxy:

adb shell settings put global http_proxy 10.0.2.2:8080

4.5 Verify proxy:

adb shell settings get global http_proxy
5. Configure Burp Suite

5.1 Open Burp Suite
5.2 Go to Proxy → Options
5.3 Under Proxy Listeners, click Edit
5.4 Select “All interfaces”
5.5 Click OK
5.6 Turn Intercept ON

6. Capture HTTP Traffic

6.1 Open Chrome in emulator
6.2 Visit:

http://example.com

6.3 Open Burp Suite → Proxy → HTTP History
6.4 Observe HTTP requests and responses

7. Export CA Certificate

7.1 In Burp Suite, go to Proxy → Options
7.2 Click Import/Export CA Certificate
7.3 Select “Certificate in DER format”
7.4 Save file as burpcer.der

8. Transfer Certificate to Emulator

8.1 Open Command Prompt
8.2 Run command:

adb push C:\Users\Admin\Downloads\burpcer.der /sdcard/Download/
9. Install Certificate in Emulator

9.1 Open Settings
9.2 Go to Security & Privacy
9.3 Click Encryption & Credentials
9.4 Select Install Certificate → CA Certificate
9.5 Choose burpcer.der

10. Verify HTTPS Interception

10.1 Open browser in emulator
10.2 Visit:

https://example.com

10.3 Open Burp Suite → Proxy → HTTP History
10.4 Verify HTTPS requests and responses

Creating and Analyzing Disk Images Using dc3dd and Autopsy
Procedure
1. Open Terminal in Kali Linux

1.1 Press Ctrl + Alt + T
1.2 Terminal will open

2. Create Sample Evidence

2.1 Create a file:

echo "Cybersecurity Lab Evidence" > evidence.txt

2.2 Verify file:

ls
3. Identify Disk Partition

3.1 List storage devices:

lsblk

3.2 Note partition (e.g., /dev/sda1)

4. Install dc3dd

4.1 Install using apt:

sudo apt update
sudo apt install dc3dd -y
5. Create Practice Disk File

5.1 Create a 100MB disk:

dd if=/dev/zero of=practice_disk.dd bs=1M count=100

5.2 Format disk file:

mkfs.ext4 practice_disk.dd
6. Create Disk Image using dc3dd

6.1 Run imaging command:

sudo dc3dd if=/dev/sda1 of=/home/kali/disk_image.dd hash=md5 log=/home/kali/acquisition.log
7. Verify Disk Image

7.1 Check image file:

ls -lh /home/kali/disk_image.dd

7.2 View log file:

cat /home/kali/acquisition.log
8. Start Autopsy

8.1 Launch Autopsy:

autopsy

8.2 Open browser and go to:

http://localhost:9999/autopsy
9. Create New Case

9.1 Click Create New Case
9.2 Enter case name and details
9.3 Click Next

10. Add Disk Image

10.1 Click Add Host → Add Image
10.2 Select image file:

/home/kali/disk_image.dd

10.3 Click Next
10.4 Keep default settings and proceed

11. Analyze Evidence

11.1 Explore file system in Autopsy
11.2 Locate file evidence.txt
11.3 Check deleted files
11.4 Analyze logs and artifacts

Conducting a Security Audit and Risk Assessment
Procedure
1. Check System Information

1.1 Press Windows + R
1.2 Type command:

msinfo32

1.3 Note:

OS Version
System Type
2. Check Windows Updates

2.1 Open Settings
2.2 Go to Windows Update
2.3 Click Check for updates

3. Check Firewall Status

3.1 Search Windows Defender Firewall
3.2 Click Turn Windows Defender Firewall on or off
3.3 Verify firewall status

4. Check Antivirus Status

4.1 Open Windows Security
4.2 Go to Virus & Threat Protection
4.3 Check:

Real-time protection → ON
Last scan → Recent
5. Check Account Security

5.1 Open Settings
5.2 Go to Accounts → Sign-in options
5.3 Verify:

Password enabled
PIN / Windows Hello enabled
6. Check Installed Applications

6.1 Open Settings
6.2 Go to Apps → Installed apps
6.3 Check for:

Unknown software
Cracked tools
7. Check Startup Programs

7.1 Press Ctrl + Shift + Esc
7.2 Open Task Manager
7.3 Go to Startup tab
7.4 Disable unnecessary applications

8. Check Network Security

8.1 Open Settings
8.2 Go to Network & Internet
8.3 Check:

Wi-Fi connection status
Network type (Public / Private)
9. Check Browser Security

9.1 Open browser (Chrome / Edge)
9.2 Go to Settings → Privacy & Security
9.3 Verify:

Safe browsing enabled
Remove unknown extensions
10. Check Data Backup

10.1 Search Backup settings
10.2 Verify:

OneDrive backup
External backup configuration
11. Perform Risk Assessment

11.1 Identify assets (files, system, network, account)
11.2 Identify threats (data loss, malware, hacking)
11.3 Identify vulnerabilities (no backup, weak password)
11.4 Assign risk level (High / Medium)
11.5 Suggest solutions

12. Apply Security Recommendations

12.1 Enable firewall
12.2 Enable antivirus
12.3 Update system regularly
12.4 Use strong passwords
12.5 Avoid unknown downloads
12.6 Practice safe browsing

13. Record Results

13.1 Note identified vulnerabilities
13.2 Record mitigation strategies
13.3 Summarize system security status
